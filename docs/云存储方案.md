# 图片上传云存储方案

## 概述

当前平台图片直接上传到服务器本地，可能存在以下问题：
- 服务器存储空间有限
- 不利于 CDN 加速
- 难以支撑高并发访问

建议接入云存储服务：阿里云 OSS 或腾讯云 COS

---

## 方案对比

| 功能 | 阿里云 OSS | 腾讯云 COS |
|------|-----------|-----------|
| 免费额度 | 5GB 存储 + 免费上传 | 50GB 存储 + 免费上传 |
| 计费方式 | 按存储量 + 流量 | 按存储量 + 流量 |
| CDN 加速 | 可绑定 CDN | 内置 CDN |
| SDK 支持 | Java/Node.js/Python | Java/Node.js/Python |
| 访问控制 | Bucket 策略 + RAM | Bucket 策略 + CAM |

---

## 推荐方案：腾讯云 COS

### 优势
1. **免费额度大**：50GB 存储 + 每月 10TB 免费流量
2. **内网访问**：与腾讯云服务器同区域可内网传输，免费
3. **一键配置**：控制台图形化配置，无需编写策略
4. **CDN 集成**：可开启 CDN 加速，提升全国访问速度

## 接入步骤

### 1. 开通腾讯云 COS

1. 访问 [腾讯云 COS 控制台](https://console.cloud.tencent.com/cos5)
2. 开通对象存储服务
3. 创建存储桶（Bucket）：
   - 名称：`linmeng-assets`
   - 地域：选择离服务器最近的区域（如广州或上海）
   - 访问权限：公有读私有写

### 2. 获取密钥

1. 访问 [密钥管理](https://console.cloud.tencent.com/cam/capi)
2. 创建 SecretId 和 SecretKey
3. 配置 Bucket 策略允许上传

### 3. 后端配置

```javascript
// server/src/config/cos.js
module.exports = {
  secretId: process.env.COS_SECRET_ID || 'your-secret-id',
  secretKey: process.env.COS_SECRET_KEY || 'your-secret-key',
  bucket: 'linmeng-assets-1234567890',  // 存储桶名称
  region: 'ap-guangzhou',                 // 区域
  baseUrl: 'https://linmeng-assets-1234567890.cos.ap-guangzhou.myqcloud.com'
}
```

### 5. 创建上传服务

```javascript
// server/src/services/cosUploadService.js
const COS = require('cos-nodejs-sdk-v5')
const config = require('../config/cos')
const path = require('path')
const crypto = require('crypto')

const cos = new COS({
  SecretId: config.secretId,
  SecretKey: config.secretKey
})

/**
 * 上传文件到 COS
 * @param {Buffer} fileBuffer - 文件内容
 * @param {string} originalName - 原始文件名
 * @param {string} folder - 存储目录，如 'resources'、'avatars'
 * @returns {Promise<string>} 文件访问 URL
 */
async function uploadFile(fileBuffer, originalName, folder = 'uploads') {
  // 生成唯一文件名
  const ext = path.extname(originalName)
  const filename = `${Date.now()}-${crypto.randomBytes(4).toString('hex')}${ext}`
  const key = `${folder}/${filename}`

  return new Promise((resolve, reject) => {
    cos.putObject({
      Bucket: config.bucket,
      Region: config.region,
      Key: key,
      Body: fileBuffer,
      ContentType: getMimeType(ext)
    }, (err, data) => {
      if (err) return reject(err)
      resolve(`${config.baseUrl}/${key}`)
    })
  })
}

/**
 * 批量上传多个文件
 */
async function uploadFiles(files, folder = 'uploads') {
  const results = []
  for (const file of files) {
    const url = await uploadFile(file.buffer, file.originalname, folder)
    results.push(url)
  }
  return results
}

/**
 * 删除 COS 文件
 */
async function deleteFile(fileUrl) {
  const key = fileUrl.replace(config.baseUrl + '/', '')
  return new Promise((resolve, reject) => {
    cos.deleteObject({
      Bucket: config.bucket,
      Region: config.region,
      Key: key
    }, (err, data) => {
      if (err) return reject(err)
      resolve(data)
    })
  })
}

function getMimeType(ext) {
  const mimeTypes = {
    '.jpg': 'image/jpeg',
    '.jpeg': 'image/jpeg',
    '.png': 'image/png',
    '.gif': 'image/gif',
    '.webp': 'image/webp',
    '.pdf': 'application/pdf'
  }
  return mimeTypes[ext.toLowerCase()] || 'application/octet-stream'
}

module.exports = {
  uploadFile,
  uploadFiles,
  deleteFile
}
```

### 6. 修改上传接口

```javascript
// server/src/routes/upload.js (新建)
const express = require('express')
const multer = require('multer')
const { uploadFile, uploadFiles } = require('../services/cosUploadService')
const { success, error } = require('../utils/response')

const router = express.Router()
const upload = multer({
  storage: multer.memoryStorage(),
  limits: { fileSize: 10 * 1024 * 1024 } // 10MB
})

// 单文件上传
router.post('/single', upload.single('file'), async (req, res) => {
  try {
    if (!req.file) return error(res, '请选择文件', 400)
    const url = await uploadFile(req.file.buffer, req.file.originalname, req.body.folder || 'uploads')
    success(res, { url })
  } catch (e) {
    console.error('Upload error:', e)
    error(res, '上传失败')
  }
})

// 多文件上传
router.post('/multiple', upload.array('files', 9), async (req, res) => {
  try {
    if (!req.files || req.files.length === 0) return error(res, '请选择文件', 400)
    const urls = await uploadFiles(req.files, req.body.folder || 'uploads')
    success(res, { urls })
  } catch (e) {
    console.error('Upload error:', e)
    error(res, '上传失败')
  }
})

module.exports = router
```

## 临时方案：本地存储优化

如果暂时不想接入云服务，可以优化本地存储：

### 当前问题
- 文件存储在 `server/uploads/`
- 无 CDN 加速

### 优化措施
1. **配置 Nginx 静态文件缓存**
2. **分离静态资源域名**（可选）
3. **开启 Gzip 压缩**

```nginx
# /etc/nginx/conf.d/linmeng.conf
server {
    listen 80;
    server_name static.3qall.com;  # 单独静态域名

    location /uploads/ {
        alias /var/www/linmeng/server/uploads/;
        expires 30d;  # 缓存 30 天
        add_header Cache-Control "public, immutable";
    }
}
```
