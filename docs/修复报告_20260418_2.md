### 问题1：商家发布资源SQL错误

**错误信息**:
```
You have an error in your SQL syntax; check the manual that corresponds to MySQL server version for the right syntax to use near '?)' at line 9
```

**根本原因**:
- INSERT语句中字段数量（35个）与VALUES占位符数量（**36个**）不匹配
- VALUES行末尾多了一个问号 `?, ?)` 而不是 `)?`

**修复方案**:
1. 修正 `merchantController.js` 中 createResource 函数的 VALUES 行：
   ```sql
   -- 修正前（36个问号）
   VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
   
   -- 修正后（35个问号）
   VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
   ```

2. 同步数据库字段：
   - 服务器数据库已有所有必需字段（之前已添加）
   - 本地数据库补充缺失字段（manpower_desc, media_desc, space_area, capacity, facilities, open_hours, work_type, salary_range）

**验证结果**:
```json
{
  "code": 200,
```

### 问题2：大使个人资料获取500错误

**错误信息**:
```
{"code":500,"message":"获取个人资料失败"}
```

**根本原因**:
- `ambassadorController.js` 的 getProfile 函数 SELECT 了不存在的 `avatar` 字段
- 服务器 ambassadors 表没有 avatar 列

**修复方案**:
```javascript
// 修正前（包含不存在的avatar字段）
SELECT id, username, real_name, phone, avatar, qr_code, status, id_card, created_at FROM ambassadors WHERE id = ?

// 修正后（移除avatar字段）
SELECT id, username, real_name, phone, qr_code, status, id_card, created_at FROM ambassadors WHERE id = ?
```

**验证结果**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 20,
    "username": "18800000003",
    "real_name": "Test Ambassador",
    "phone": "18800000003",
    "qr_code": "LM88888888",
    "status": 1
  }
}
```
